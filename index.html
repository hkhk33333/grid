DER Hosting Capacity Planner

⚡
A utility planner asks: "Can Feeder 4012 support 200 new rooftop solar installations and a community battery?"
Today, that question takes a manual engineering study. With the Hosting Capacity Planner, it takes minutes — powered by GPT-D's simulation engine and Tapestry's knowledge graph.

What This Requires

  • GPT-D's distribution simulation engine for feeder-level power flow
  • Tapestry's knowledge graph for asset topology and condition data
  • Thermal, voltage, and protection limit modeling per circuit
  • Visual map layer showing capacity by location (green/yellow/red)

Why Now

  • GPT-D already deployed with Vector NZ — simulation foundation exists
  • DOE's DER Interconnection Roadmap (Jan 2025) calls for hosting capacity transparency
  • Data center boom is accelerating distribution-level load growth
  • GridAware asset data feeds directly into capacity models
Competitive pressure: Academic tools like PowerChain (2025) demonstrate agentic AI can automate distribution analysis. If Tapestry doesn't productize hosting capacity, third parties will — without Tapestry's data moat.

Technical Architecture

Input
Utility uploads DER siting request
→
Data Layer
Knowledge graph resolves feeder topology
→
Simulation
GPT-D runs power flow + contingency analysis
→
Output
Hosting capacity map + upgrade recommendations

Latency

Challenge
Full feeder simulation is compute-intensive. Use pre-computed base cases with incremental delta analysis for sub-minute responses; full re-sim on demand.

Cost

Challenge
Simulation at scale requires significant compute. Tiered approach: static maps refresh nightly; dynamic queries use on-demand compute charged per analysis.

Failure Handling

Challenge
Incomplete utility data leads to unreliable results. Flag confidence levels per feeder; require minimum data thresholds before showing capacity estimates.

Privacy

Challenge
Feeder-level data can reveal customer information. Aggregate to circuit level for public maps; utility-only views for granular analysis behind auth.

Monthly queries per utility over time — increasing depth signals the tool is replacing manual workflows, not just supplementing them.

Why These Metrics

Grid software adoption is slow and high-stakes — utilities won't switch from manual studies unless the tool proves reliable over repeated use. Adoption measures go-to-market traction. Study time reduction measures whether the tool delivers on its core promise (validated against the utility's own manual study benchmarks). Query depth is the leading indicator of retention: if planners run more analyses month-over-month, the tool is becoming part of their workflow, not a novelty.
